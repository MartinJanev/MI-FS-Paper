# MI-FS Benchmark

Mutual-information feature-selection benchmark for tabular datasets. The project runs multi-seed cross-validation, aggregates results, and produces paper-ready plots and tables.

## Start here

If you just cloned the repo, do this in order:

1. Create a virtual environment and install the package.
2. Prepare the processed dataset folds.
3. Run the experiment script for one dataset.
4. Aggregate the raw outputs.
5. Generate figures, tables, or confusion-matrix evidence if needed.

## What you need first

- Python environment with the project installed in editable mode.
- Raw source files under `src/data/raw/<dataset>/` before preprocessing.
- For Santander, Home Credit, and IEEE-CIS Fraud, download the original Kaggle competition files first.
- Arcene uses the ARCENE benchmark files in `src/data/raw/arcene/`.
- Processed data under `src/data/processed/<dataset>/` is generated by the preparation scripts.
- A config file from `src/configs/`.
- Enough disk space for raw and aggregated CSV outputs.

## The shortest full workflow

### 1) Prepare the data once per dataset

Run the matching preparation script for the dataset you want to benchmark:

```powershell
python src/mi_fs_benchmark/scripts/preparation/prepare_santander.py
python src/mi_fs_benchmark/scripts/preparation/prepare_home_credit.py
python src/mi_fs_benchmark/scripts/preparation/prepare_ieee_fraud.py
python src/mi_fs_benchmark/scripts/preparation/prepare_arcene.py
```

This creates files like `fold_0.npz`, `fold_1.npz`, ... and `metadata.json` in `src/data/processed/<dataset>/`.

### 2) Run the experiment

Main runner: `src/mi_fs_benchmark/scripts/run_experiment.py`

```powershell
python src/mi_fs_benchmark/scripts/run_experiment.py --config src/configs/santander.yaml --n-runs 10 --use-gpu --device cuda:0
```

If you do not want GPU acceleration, add `--no-gpu`.

Raw run outputs go to `src/results/raw/` with names like:

- `santander__run001__seed236040.csv`

### 3) Aggregate the runs

Single dataset:

```powershell
python src/mi_fs_benchmark/scripts/aggregate_results.py --dataset santander
```

Multiple datasets:

```powershell
python src/mi_fs_benchmark/scripts/aggregate_results.py --datasets santander,home_credit,ieee_cis_fraud
```

All datasets found in raw outputs:

```powershell
python src/mi_fs_benchmark/scripts/aggregate_results.py --all-datasets
```

Important outputs in `src/results/aggregated/`:

- `<dataset>_summary.csv`
- `<dataset>_stability_summary.csv`
- `<dataset>_statistical_significance.csv`
- `combined_summary.csv`
- `combined_stability_summary.csv`
- `combined_statistical_significance.csv`
- `combined_table_source.csv`

### 4) Make figures and tables

Primary plotting script:

```powershell
python src/mi_fs_benchmark/scripts/make_paper_plots.py --root . --outdir paper_figs
```

Alternative script that also writes the reviewer-style summary table:

```powershell
python src/mi_fs_benchmark/scripts/make_paper_plots_2.py --summary_csv src/results/aggregated/combined_summary.csv --stability_csv src/results/aggregated/combined_stability_summary.csv --table_source_csv src/results/aggregated/combined_table_source.csv --outdir paper_figs
```

### 5) Generate confusion-matrix evidence when F1 looks suspiciously low

If you need proof that a very small F1 score is real, generate fold-level and aggregated confusion-matrix evidence:

```powershell
python src/mi_fs_benchmark/scripts/generate_confusion_matrices.py --dataset home_credit
```

This writes:

- `src/results/aggregated/home_credit_confusion_fold_level.csv`
- `src/results/aggregated/home_credit_confusion_by_setting.csv`
- `paper_figs/fig_confusion_home_credit_<model>_kstar_peak.png`

## Where to look for the main code

- `src/mi_fs_benchmark/scripts/`: run, aggregate, plot, and preparation entry points.
- `src/mi_fs_benchmark/core/fs/`: feature selectors.
- `src/mi_fs_benchmark/core/models/`: downstream models.
- `src/mi_fs_benchmark/experiment/`: cross-validation, evaluation, and metrics.
- `src/configs/`: dataset experiment configs.
- `src/data/`: raw and processed data.
- `src/results/`: raw and aggregated outputs.

## How to read the outputs

- The shaded region in the plots is the confidence interval band around the mean, usually 95% CI.
- In the summary table, the `*` marker means the selector is flagged by the primary paired t-test against the best baseline in the generated significance output; the Wilcoxon result is stored separately.
- Confusion-matrix plots are arranged for readability as `TP` top-left, `FN` top-right, `FP` bottom-left, `TN` bottom-right.

## Available configs

The repository includes these configs in `src/configs/`:

- `santander.yaml`
- `home_credit.yaml`
- `ieee_fraud.yaml`
- `arcene.yaml`

Typical config fields:

```yaml
dataset:
  name: santander
  root: src/data/processed/santander

selectors: [variance, anova, l1_logreg, tree_importance, mi, mrmr, boruta, shap]
models: [logreg, hgbt]
k_values: [5, 10, 20, 35, 50, 75, 100, 150]
cv_outer_folds: 5
```
